RemitChain smart contract blueprint — three Solidity contracts with function signatures

RemitChain.sol — main router

core

Entry point for all transfers. Coordinates KYC check → escrow lock → notification event.

sendRemittance()Sender calls with recipient address/phone, amount in QUSD, destination country. Checks KYC level, locks escrow, emits event.
claimRemittance()Recipient calls with transfer ID + OTP hash. Validates, triggers escrow release.
cancelRemittance()Sender can cancel if unclaimed after 48h. Escrow refunds automatically.
getTransferStatus()Read-only. Returns PENDING / CLAIMED / CANCELLED for any transfer ID.

EscrowVault.sol

money

Holds QUSD in trust. Only RemitChain.sol can instruct it.

lockFunds()Pulls QUSD from sender, mints a transfer receipt NFT as proof.
releaseFunds()Sends QUSD to recipient minus 0.1% fee. Only callable by RemitChain.
refundFunds()Returns full amount to sender on cancel. Burns receipt NFT.

Transfer states:

PENDING — funds locked
CLAIMED — released to recipient
CANCELLED — refunded to sender

KYCRegistry.sol

identity

Maps QIE Pass IDs to KYC levels and daily limits.

verifyUser()Links wallet address to QIE Pass ID. Sets KYC tier (0/1/2).
getKYCLevel()Returns tier for any address. Level 0 = no KYC, Level 2 = full ID.
getDailyLimit()Returns QUSD limit. Tier 0: $0, Tier 1: $500/day, Tier 2: $5000/day.
checkDailyUsage()Prevents breach of daily limit. Resets at midnight UTC.

Key design decisions that impress judges

Reentrancy guard on all fund-moving functions (use OpenZeppelin ReentrancyGuard)
No floating point — all amounts in smallest QUSD unit (6 decimal places)
Events emitted on every state change — your frontend listens via WebSocket
48-hour auto-expiry on unclaimed transfers — safe for the sender
